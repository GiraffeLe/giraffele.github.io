第一章:

* 网络信息安全的目标:

  • 机密性(Confidentiality) : 信息不被非授权访问,即便被非法访问到了也无法知晓信息内容
  • 完整性(Integrity): 信息在生成传输存储过程中不被篡改(人为和非人为)
  • 抗否认性(Non-repudiation) : 用户无法在事后否认对信息操作的行为(生成签发,接收等)
  • 可用性(Availability) :

密码学理论:(Cryptography)

包含以下几点:

数据加密 Data Encryption

数据加密算法是一种数学变换. 将信息从 易于理解的明文加密为 不易理解的密文

根据加密解密的密钥种类分为两种:

对称算法: 加解密密钥相同的算法

例如: DES,AES

非对称算法: 加解密 密钥不同的算法

例如: RSA,ECC

信息摘要:(Message Digest)

也是一种数学变换 通常是单向的

将不定长度的信息变换为固定长度的摘要

信息的任何改变(即使是1b)都会引起摘要面目全非

经典算法:

MD5,SHA

数字签名(Data Signature)

消息摘要 + 非对称加密算法的组合应用

密钥管理:

主要内容是 密钥的产生,发放,存储,更换和销毁的算法和协议等.

身份认证:

验证用户身份与其所生成的身份是否一致.

常见的是口令认证.

授权和访问控制

审计和追踪

安全协议:

典型的有:

网络层安全协议 IPSec ,传输层安全协议 SSL

应用层安全电子商务协议SET

第二章:

密码学基础:

基本概念:

加密: 对需要保密的消息进行编码

加密算法: 编码的规则

明文: 需要加密的消息

密文: 明文加密之后的形式

解密: 把密文恢复成明文的过程

解密算法 解密的规则

单密钥系统

又称对称密码系统

特点: 加密密钥和解密密钥相同 或者 实质上等同(从一个得出另一个)

加密方式:

对明文按字符

逐位加密 => 流密码/序列密码

先分组,后逐组加密 => 分组密码

双密钥系统

又称非对称密码系统/公开密钥密码系统

双密钥系统中有两个密钥 一个公开的 K1 (谁都可以用) 另一个是私人的 K2 (只有采用此系统的人才可以用)

注:

从公开的密钥推不出私人密钥

公钥加密实现公共网络上的保密通信

经典密码体制:

单表代换密码:

就是字母 a~z 的自然顺序不变.

每个字符都按照一个固定的偏移量进行替换.例如偏移量为3,那么明文中的每个字母都往右移动3个位置.

多表代换密码:

多表代换密码使用多个不同的替换规则，而不是仅仅依赖于一个固定的替换表。

多字母代换密码:

Hill密码/希尔密码

分组密码原理:

分组密码是按照一定长度例如64bit,128bit 对明文进行加密.

然后以组为单位进行加解密.

设计原理:

分组密码的一般结构:

分为两种: Feistel网络结构和SP网络结构

Feistel网络结构:

SP网络结构

数据加密标准:

DES (Data Encryption Standard) 数据加密标准

使用最广泛的加密算法.

DES描述:

DES是分组长度为64b 的分组密码算法.密钥长度也是64b,其中每8b有一位奇偶校验位,所以实际有效长度位56b

DES算法是公开的.

AES 高级加密标准

四个步骤:

  1. 字节代替 BS
  2. 行移位 SR
  3. 列混淆 MC
  4. 轮密钥异或

单项散列函数:

概述:

单向散列函数 Hash Function 又称哈希函数

特点:

将任意长度的消息M映射成一个固定长度散列值h的函数H

即$ h = H(M)$

特性:

给定M 容易得出 h

给定 h 反推 M 很难

只有 两个M 相同, 两个h才会相同.

MD5算法

SHA-1算法

二者比较

特征项 SHA-1 MD5
Hash值长度 160b 128b
分组处理长度 512b 512b
步数 80(4*20) 64(4*16)
最大消息长度 <=2^64^b 不限

SHA1比MD5安全

MD5比sha1快
